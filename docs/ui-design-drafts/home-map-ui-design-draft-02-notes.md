# 首页高保真设计稿说明

对应图片：`home-map-ui-design-draft-02.png`

## 定位

这是 Link Match 连连看首页/章节地图的高保真视觉目标稿，用于后续开发首页 UI。设计延续主游戏页的水果森林、奶油果冻面板、青绿色标题牌和高饱和水果糖果质感。

## 主尺寸

- 主稿按 `390 x 844` CSS 像素比例设计。
- 当前生成图片为高清等比例输出，实际文件尺寸为 `853 x 1844`，实现时按比例还原到移动端视口。

## 页面结构

- 顶部固定区：个人入口、设置入口、体力、金币、星星资源栏。
- 章节切换区：水果森林、糖果花园、果冻城堡三段式章节 tab，并保留左右切换按钮。
- 中部弹性区：水果森林关卡地图，蜿蜒果冻路承载关卡节点。
- 底部固定区：`开始闯关` 主按钮，需留出底部安全区。

## 状态规则

- 初始状态只开放第 1 关。
- 当前关使用粉色发光大节点。
- 已解锁未通关关卡使用青绿色节点，可显示空星槽。
- 已通关关卡显示历史最高星级。
- 未解锁关卡使用蓝色或低饱和节点，并显示锁定感。
- 设计稿中的星级表现用于展示视觉样式，开发实现时以实际本地进度数据为准。

## 长宽屏适配

- 顶部资源栏和底部按钮高度相对固定。
- 多出来的屏幕高度优先给中部地图区域，长屏手机可多显示关卡点。
- 内容最大宽度建议限制在 `430px` 左右，宽屏手机居中显示，不横向拉伸地图。
- 底部按钮使用 `safe-area-inset-bottom` 适配 iPhone Home Indicator 和安卓手势区。

## 后续实现重点

- 首页背景、章节 tab、关卡节点、主按钮需要从高保真稿中拆分或重新生成独立切图。
- 关卡路径不要整张截图贴死，应由地图容器、路径、节点和装饰层组合实现。
- 后续如果生成项目图片资产，继续使用 `gpt-image-2`。
