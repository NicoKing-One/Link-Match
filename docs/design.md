# Link Match 连连看设计说明

## 1. 设计方向

当前视觉方向为“水果糖果风 + 章节地图 + 清晰棋盘”。整体应保持轻量、明亮、圆润、适合移动端竖屏单手操作，同时避免回退到普通 HTML 卡片感。

核心气质：

- 首页是开放式章节地图，不使用中部地图奶油边框。
- 三主题分别用水果森林、糖果花园、果冻城堡建立视觉差异。
- 游戏页保持高饱和果冻花园背景、清晰棋盘和底部大圆道具按钮。
- 弹框和结果页统一为水果糖果果冻风，使用同一套卡片、按钮和顶部徽章语言。

## 2. 页面结构

首页：

- 顶部：左侧个人入口、右侧设置入口。
- 资源栏：体力、金币、星星三张资源卡，资源 icon 与文字分区展示。
- 章节区：左右箭头切换水果森林、糖果花园、果冻城堡。
- 地图区：每个主题显示 30 个关卡节点，01 关贴近底部，30 关贴近顶部，左右交替排布。
- 左侧兑换入口：保留独立图片入口，当前只弹“功能暂未开放”。
- 底部：开始闯关按钮。

二级页：

- 个人中心和设置页均由 HTML/CSS 组件化布局组成，不使用整页设计图铺底。
- 二级页只保留左上返回按钮，不再增加底部返回首页按钮。

游戏页：

- 顶部 HUD：关卡、时间、剩余、得分、体力。
- 中部棋盘：CSS Grid 动态布局，按当前关卡 rows/cols 调整。
- 底部工具栏：提示、洗牌、暂停三个大圆工具按钮。
- 返回首页入口触发确认弹框，不直接离开本局。

弹框/结果页：

- 暂停、离开本局、道具用完、体力不足、通关成功、挑战失败使用统一果冻弹框体系。
- 结果页在弹框后有黑灰色蒙版。
- 结果页不再显示体力栏、得分/最佳分统计块和标题下方说明文。

## 3. 首页视觉规范

- 主容器移动端竖屏优先，最大宽度保持 520px。
- 首页关键尺寸基于容器宽度缩放，窄屏应保持内容不挤压、不重叠。
- 资源卡采用 40% icon 区 / 60% 文案区：
  - icon 尺寸约 `8cqw`，贴近 icon 区右侧。
  - 文案字号约 `4cqw`，贴近文案区左侧，并保留 `margin-left: 1cqw`。
- 三主题背景使用 `src/assets/image/background-*-full.png` 完整背景。
- 关卡节点统一为三层结构：
  - 顶部星级徽章：`road-stars-1.png`、`road-stars-2.png`、`road-stars-3.png`。
  - 中部状态图：已通关、当前、未开始三态主题资源。
  - 底部数字牌：HTML 文本渲染，保持居中。
- 水果森林连接线使用藤曼图片分段渲染。
- 糖果花园连接线使用粉白糖果棒图片分段渲染。
- 果冻城堡连接线使用冰晶/水晶链图片分段渲染。
- 当前关卡保留轻微呼吸动画，动画不得改变布局尺寸。

## 4. 游戏页视觉规范

- 游戏页使用 `page-bg-saturated-jelly-v2.png` 作为当前主背景。
- `.board-wrap` 保留浅色实线边框，不再添加厚重背景或阴影覆盖。
- 棋盘格由 CSS Grid 生成，尺寸由关卡配置动态决定。
- 棋子使用 `src/assets/image/` 下的水果糖果图片资源，图标应占格子主体区域，避免过小。
- 连线路径使用 SVG 圆角 `path` 绘制，短暂显示后消失。
- HUD 图标和文字应紧凑、清晰，适合 320/360/390 宽度视口。
- 底部提示/洗牌角标使用固定尺寸徽标，不能因数字变化导致按钮跳动。

## 5. 弹框与结果页规范

- 弹框背景使用 `src/assets/image/modal-card-bg.png`。
- 主按钮使用 `src/assets/image/modal-button-primary.png`。
- 副按钮使用 `src/assets/image/modal-button-secondary.png`。
- 暂停弹框顶部徽章使用 `modal-pause-badge.png`。
- 离开本局弹框顶部徽章使用 `modal-exit-badge.png`。
- 通关成功顶部徽章使用 `result-pass-badge.png`。
- 挑战失败顶部徽章使用 `result-fail-badge.png`。
- 金币奖励图标使用 `result-coin.png`。
- 顶部徽章一半压在弹框外、一半压在弹框内，徽章内动态展示当前关卡，如 `第01关`。
- 结果页按钮间距统一，首通成功按钮顺序为双倍金币、下一关、再玩一局、返回首页。
- 失败结果页按钮顺序为复活、再玩一局、返回首页；购买道具入口为占位。
- 结果页反馈 toast 位于结果卡片内，不漂到卡片外。

## 6. 交互状态

- 选中：图案上浮并出现金色描边。
- 可提示：一对图案脉冲高亮，并显示连接线。
- 非法选择：显示原因、选中图案短暂抖动，并清空选中状态。
- 消除：显示连线，随后两个图案透明消失。
- 暂停：倒计时停止，棋盘状态保留。
- 道具耗尽：弹出道具用完弹框，看广告/购买按钮当前提示“功能暂未开放”。
- 体力不足：弹出体力不足弹框；看广告获取可加 30 点体力，购买体力当前提示“功能暂未开放”。
- 未开放入口：统一提示“功能暂未开放”，包含首页兑换入口、购买体力、购买道具等。
- 音频：按钮反馈使用统一短音；成功、失败、奖励使用独立反馈；设置页开关只影响当前会话。

## 7. 资源规范

- 所有运行时图片统一放在 `src/assets/image/` 根目录，不再使用图片子目录。
- 所有新生成图片必须使用 `gpt-image-2`。
- 工具、关卡节点、连接线等资源应生成独立 PNG，不从大图切 sprite。
- 透明资源必须紧贴可见内容裁切，不保留透明 padding 或空白边。
- 背景图应使用完整图，不从截图或设计稿中截取。
- 本地后处理只做透明化、裁边、等比缩放、居中贴底、内部透明洞封闭等必要清理，不做非等比拉伸。
- `dist/` 和 `output/` 只作为构建或 smoke 临时产物，不作为仓库资源保留。

## 8. 可访问性与验证约束

- 关键按钮必须保留可读 `aria-label` 或可见文本。
- 结果页状态反馈使用 `role="status"` / `aria-live` 口径。
- 锁定关卡不可点击，当前/已通关关卡状态需要明确。
- 文案在 320/360/390 宽度下不能相互遮挡或溢出按钮容器。
- smoke 断言覆盖的布局和数据口径应保持稳定：首页资源卡比例、三主题连接线、当前关卡居中、二级页组件结构、设置开关、个人中心统计、结果页按钮和未开放文案。

## 9. 设计后续

- 真机继续复核三主题地图滑动手感、窄屏文字、节点遮挡和连接线风格。
- 若替换二级页正式 icon 或图片，保持当前 HTML/CSS 结构不变，只替换 `src/assets/image/` 下对应资源。
- 若重新加入清除本地进度入口，需要先定义入口位置、确认弹框和清档后的反馈样式。
- 若推进商城/头衔系统，需要先补齐拥有、佩戴、金币扣除、重复点击和持久化状态，再设计正式页面。
