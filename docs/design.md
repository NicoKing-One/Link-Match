# 连连看 MVP 设计说明

## 1. 设计方向

视觉方向选择“轻量休闲 + 清晰棋盘”。页面优先服务重复游玩：按钮固定、信息密度适中、棋盘区域最大化。

## 2. 页面结构

- 首页：品牌区、难度选择、开始按钮。
- 游戏页：HUD、棋盘、操作栏。
- 暂停弹窗：暂停状态说明、继续按钮。
- 结算页：结果标题、摘要、得分、最佳分、再玩一局、返回首页。

## 3. UI 规范

- 移动端优先，主容器最大宽度 520px。
- 卡片圆角不超过 8px，避免过度装饰。
- 棋盘使用 CSS Grid，尺寸由难度配置动态决定。
- 图案使用字符符号和颜色区分，不生成图片文件。
- 连线路径使用 SVG `polyline` 绘制，短暂显示后消失。

## 4. 交互状态

- 选中：图案上浮并出现金色描边。
- 可提示：一对图案脉冲高亮，并显示连接线。
- 非法选择：新选中的图案短暂高亮，原选择切换为新图案。
- 消除：显示连线，随后两个图案透明消失。
- 暂停：倒计时停止，棋盘状态保留。

## 5. 工具建议

自己操作：

- 用纸笔或白板画首页、游戏页、结算页。
- 用 Markdown 写简版 PRD。
- 用浏览器试玩和记录问题。

插件/工具更快：

- Product Design：从文字想法生成 3 个视觉方向。
- Figma：沉淀高保真 UI、组件、间距、颜色和开发交付。
- Browser / Playwright：做浏览器自动化验收和截图检查。
- Documents：把 PRD 输出成 Word 或更正式的交付文档。
- Figma Dev Mode：给前端查看规格、资源、代码片段和版本差异。

图片规则：

- 如果后续需要生成卡牌、背景、图标、宣传图，必须使用 `gpt-image-2`。
