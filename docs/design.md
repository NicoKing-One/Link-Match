# 连连看 MVP 设计说明

## 1. 设计方向

视觉方向选择“水果糖果风 + 清晰棋盘”。当前主界面以用户指定参考图为高保真视觉目标：柔和水果花园背景、透明玻璃棋盘托盘、奶油 HUD、绿色果冻标题牌和底部大圆工具按钮。

## 2. 页面结构

- 首页：品牌区、难度选择、开始按钮。
- 游戏页：顶部关卡编号横幅、奶油 HUD、透明玻璃棋盘托盘、底部奶油道具栏。
- 暂停/道具/离局/体力弹窗：统一果冻卡片、顶部徽章图标、说明文案和操作按钮。
- 结算页：结果徽章、结果标题、摘要、星级、得分、最佳分、体力入口、再玩一局、返回首页。

## 3. UI 规范

- 移动端竖屏优先，主容器最大宽度 520px。
- 页面使用水果糖果背景图和半透明奶油/果冻面板。
- 主游戏页必须优先对齐用户给出的 UI 参考局部图，不要再只按普通 HTML 卡片风格修饰。
- 游戏页 HUD 使用 CSS 绘制的小图标区分时间、剩余、得分、体力。
- 底部道具按钮使用大圆形果冻按钮，提示/洗牌次数使用角标。
- 弹窗和结算页使用统一顶部徽章，避免每个状态各自一套视觉语言。
- 棋盘使用 CSS Grid，尺寸由难度配置动态决定。
- 图案使用 `src/assets/image/` 下的水果糖果图片资源，图标约占格子 90%。
- 连线路径使用 SVG 圆角 `path` 绘制，短暂显示后消失。

## 4. 交互状态

- 选中：图案上浮并出现金色描边。
- 可提示：一对图案脉冲高亮，并显示连接线。
- 非法选择：提示浮层显示原因，两个选中图案短暂抖动，并清空选中状态。
- 消除：显示连线，随后两个图案透明消失。
- 暂停：倒计时停止，棋盘状态保留。
- 道具耗尽：弹出看广告获取或购买占位弹窗。
- 体力不足：弹出看广告获取体力、购买体力、稍后再说。

## 5. 工具建议

自己操作：

- 用纸笔或白板画首页、游戏页、结算页。
- 用 Markdown 写简版 PRD。
- 用浏览器试玩和记录问题。

插件/工具更快：

- Product Design：从文字想法生成 3 个视觉方向。
- Figma：沉淀高保真 UI、组件、间距、颜色和开发交付。
- Browser / Playwright：做浏览器自动化验收和截图检查。
- Documents：把 PRD 输出成 Word 或更正式的交付文档。
- Figma Dev Mode：给前端查看规格、资源、代码片段和版本差异。

图片规则：

- 如果后续需要生成卡牌、背景、图标、宣传图，必须使用 `gpt-image-2`。
