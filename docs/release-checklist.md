# 连连看发布检查清单

## 1. 开发前准备

- 初始化 Git 仓库，确保每个可运行版本都有提交记录。
- 冻结 `docs/PRD.md` 和 `docs/design.md` 的 MVP 范围。
- 确认目标平台：先 H5 预览，再选择微信小游戏或抖音小游戏适配。
- 准备平台账号、主体资质、游戏名称、图标、简介、隐私政策。

## 2. 测试流程

- 规则单测：运行 `node --test tests/*.test.mjs`。
- 浏览器测试：启动本地服务，分别测试 390×844 和 1440×900 视口。
- 真机测试：扫码或局域网访问，检查点击、暂停、重开、结算。
- 回归场景：合法连接、非法连接、提示、洗牌、时间到、通关、最佳分。

## 3. 打包流程

- H5 预览包：运行 `scripts/build.mjs`，输出 `dist/`。
- 检查 `dist/index.html`、`dist/game.js`、`dist/styles.css` 是否存在。
- 上线前清理调试日志、确认版本号、确认入口页面。
- 若要转小游戏，使用平台适配工程承载 `src/` 中的游戏逻辑。

## 4. 微信小游戏发布

- 安装微信开发者工具。
- 创建或导入小游戏项目，填写 AppID。
- 在开发者工具中预览和真机调试。
- 上传体验版，邀请测试成员验证。
- 在微信公众平台提交审核。
- 审核通过后手动发布。

需要你自己操作：

- 注册/认证账号、主体资质、AppID、支付/广告资质、隐私合规确认。

我或插件更快：

- 生成发布说明、隐私政策草稿、测试清单、驳回原因修复方案。

## 5. 抖音小游戏发布

- 注册抖音开放平台账号并创建小游戏。
- 完善基础信息、资质和备案材料。
- 使用抖音开发者工具导入项目并调试。
- 上传测试版本，扫码真机验证。
- 提交审核。
- 审核通过后选择灰度发布或全量发布。

需要你自己操作：

- 平台认证、资质上传、备案、审核提交和最终发布确认。

我或插件更快：

- 准备版本说明、测试账号说明、审核材料清单、灰度发布回滚清单。

## 6. V2 后端上线建议

- 先接云存档和排行榜，不急着接广告/支付。
- 每个接口先写数据结构和验收标准，再开发。
- 平台登录态、用户授权、排行榜和广告回调必须在真机环境测试。
